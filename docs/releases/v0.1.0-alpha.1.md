# Codex Sparkompass v0.1.0-alpha.1

Diese Alpha bereitet den nächsten GitHub-Release vor: klar belegte Token-Ersparnis, aber mit stärkerem Qualitätsnetz statt aggressiver Blindkompression.

## Was ist neu?

- Neuer Schutzfall für Web-Security-Header und Cookies.
- Neue kritische Ankerklasse `web-security-header`.
- `Set-Cookie`, `HttpOnly`, `Secure`, `SameSite`, CORS-Origin, CSP-Nonce, `frame-ancestors` und CSRF-Verträge bleiben exakt erhalten.
- Benchmark jetzt mit `26` Fällen und `23/23` Failure-Corpus-Klassen.
- Gegenfakten-Prüfung jetzt `169/169` erkannt.
- Versionen in `package.json`, `package-lock.json` und Plugin-Manifest synchron auf `0.1.0-alpha.1`.
- README, Evidenzseite, Release-Checklist und Einsparungsdiagramm auf denselben Messstand gebracht.

## Aktueller lokaler Belegstand

- Tests: `277/277`
- Benchmark: `26/26` Kontext-Erfolge
- Failure-Corpus: `23/23`
- Gegenfakten: `169/169`
- Benchmark-Ersparnis: `52%`
- Benchmark-Gesamtkosten: `133` Tokens pro verifiziertem Task
- Dogfood: `36%` durchschnittliche Ersparnis
- Kritische Anker: `100%`
- A/B-Beleg: `8.417` Tokens gespart, `21%` Gesamt-Ersparnis, `16.051` weniger nicht gecachte Input-Tokens, `42%` weniger nicht gecachter Input

## Warum das wichtig ist

Sparkompass spart weiterhin deutlich Kontext, aber die neue Version schützt einen weiteren realen Risikobereich: Web-Security-Konfiguration. Gerade Cookie-, CORS-, CSP- und CSRF-Zeilen dürfen nicht nur ungefähr erhalten bleiben. Kleine Verluste können Sicherheitsverhalten ändern.

## Installation

```bash
npm ci
node ./bin/codex-sparkompass.mjs doctor
node ./bin/codex-sparkompass.mjs audit .
```

Vollständige Maintainer-Verifikation:

```bash
npm run check
```

## Einschränkungen

- Technical Preview, noch keine Stable-Version.
- Noch nicht offiziell in Codex eingebaut.
- Kein automatisches Umschreiben des internen Codex-Requests.
- Tokenwerte sind lokale Messwerte oder lokale Usage-Belege, keine Abrechnung.
- `verified-end-to-end-noninferior` ist vorbereitet, aber noch nicht final erreicht.
